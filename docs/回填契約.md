# 回填契約 —— mandala 巡倉回收之制（2026-07-09 立）

> mandala（壇城）定期巡此倉，回收落筆成果入壇。此卷為兩倉交接之**唯一契約**：
> 粉本側承諾何者穩定，壇城側承諾何者不越。兩倉之 agent 皆當先讀此卷。

## 一 · 大義：閘在讀端，牒隨貨行

- **拷全庫，不揀貨**：巡倉一律整拷 `dist/`（機出，一字不改），無論其中幾尊已核。
  揀貨之智不在拷貝工序——在讀端之閘（下條）。故巡倉可無腦定期跑，冪等可重。
- **壇上只認「上壇之()」**：mandala 取尊必經
  `import { 上壇之 } from 'vendor/fenben/dist/baimiao.js'`——
  三戒同持乃返 `{ 面, 鍵 }`（儀軌信已核・落筆簿有專筆・筆非候審），否則 null。
  描之必**面鍵並傳**：`const 上 = 上壇之(id, 側); if (上) 白描(ctx, R, 上.面, 上.鍵);`
  ——鍵缺則白描退通形示意，此陷契約明鑄於返值。
  **null 者守 mandala 現行佔位**，勿逕畫候審之筆——寧缺毋誤行於壇。
- **牒隨貨行**：`dist/牒.json` 逐面錄（鍵・尊・信・筆・候審・上壇），
  巡倉讀牒即知本次新增幾尊可上壇，據以寫 commit 記錄，勿須解模塊內情。
  牒為**純內容清單，無版戳無時戳**——輸出決定於源，build 冪等；
  溯源（版・回填之時）在回填工序生成之 `vendor/fenben/來源.md`。

## 三 · 巡倉工序（mandala 側定期任務）

```
① cd fenben && git pull && npm run build        # build 綴牒（tsc && node tools/牒.cjs）；pull 後樹淨
② node tools/回填.cjs <mandala倉根>              # 先刪後整拷 dist → vendor/fenben/dist（舊檔不殘），
                                                 # 來源.md 於此時取 HEAD 為溯源之版（樹不淨則警）
③ cd mandala：煙驗——逐「上壇」鍵以 白描(ctx,R,面,鍵) 描於 stub canvas，無擲乃過
④ mandala commit，message 必錄來源.md 之版與本次新上壇之尊——溯源之責在壇側
```

- 拷貝工序之碼**住在粉本倉**（tools/回填.cjs），壇側只呼之——拷貝邏輯單源。
- 巡倉頻率隨意（冪等）；無新上壇之尊時 vendor 之 diff 仍可能有（候審筆之進展隨 dist 同行），
  壇側可照常 commit（記「無新上壇」）或跳過，皆合約。

## 四 · 升核即出貨之界

- 粉本側工序：落筆 → commit（陳列頁朱界候審）→ **主人印可** → 信升已核・
  自候審筆摘鍵 → 下次巡倉自動上壇。出貨之界＝主人之印，無旁路。
- 壇側永不催貨、不代核；粉本側永不直推 mandala 倉（回填必經巡倉工序，單向拉取）。

## 五 · 適配層之責（mandala funpon.js）

- funpon.js 為薄適配：`const 上 = 上壇之(id, 側)`，有則 `白描(ctx, R, 上.面, 上.鍵)`，
  無則走壇城現行佔位形。
- 舊名（verify／GIKI／kosareta／drawFunpon）已備於粉本，適配層勿刪其引用。
- 壇城之座標縮放（月輪半徑 R 之取值）由壇側自定，粉本格網無量綱自適。
